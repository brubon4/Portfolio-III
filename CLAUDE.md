## Структура файла index.html
Один большой HTML-файл. Разделы сверху вниз:
1. `<nav>` — навигация (скрывается при скролле вниз, появляется при скролле вверх)
2. `.hero` — главный экран (canvas с частицами, scanline, счётчики)
3. `.about` — обо мне + фото (`uploads/EC5D65A9-87C7-431A-AAC1-FFC829E0DD31.png`)
4. `.experience` — опыт работы (6 мест, хронология)
5. `.skills` — навыки (3 колонки: BI/аналитика, Проектирование/BIM, Управление/Soft)
6. `.analytics` — аналитика: стек + блок-схемы проектов (SVG с переключателем)
7. `.objects` — объекты
8. `.education` — образование
9. `.contact` — контакты (есть ссылка на GitHub)
10. `<footer>`

## Анимации и эффекты
- `@keyframes shimmer` — sweep слева направо (используется на `.hero-tag::after`, `.skill-tag::after`, `.section-tag::after`)
- `@keyframes shimmer-rare` — то же но с длинной паузой (82% цикла стоит, 14s) — для тегов навыков
- `@keyframes scandown` — горизонтальная линия сверху вниз по герою
- `@keyframes pulse` — мигание зелёной точки "доступен"
- JS: случайные задержки `--sd` для шиммера на каждом теге (Math.random * 4s)
- IntersectionObserver — scroll-reveal для карточек
- Canvas — частицы с соединительными линиями в герое

## Блок-схемы проектов (`.analytics`)
- SVG-файл: `uploads/portfolio_projects_v4.svg` (680×1160px, два проекта вертикально)
- Переключатель `.proj-switch-btn` показывает/скрывает `.svg-proj` div-ы
- Проект 1: `viewBox="0 0 680 568"`
- Проект 2: `viewBox="0 568 680 592"`
- На десктопе (≥901px): `.svg-proj { width: 50%; margin: 0 auto; }` — вдвое меньше
- На мобильном: полная ширина

## Git-workflow
- Всегда коммитить в `main` напрямую (или мержить из вспомогательной ветки)
- После пуша GitHub Pages обновляется за 5–10 минут
- Для проверки без кеша: открыть в режиме инкогнито или Ctrl+Shift+R

## Мобильная версия (≤900px)
- `section { padding: 36px 20px; }`
- Навигация скрывает часть ссылок
- Опыт работы — горизонтальный скролл с точками-индикаторами
- Блок-схемы — полная ширина

## Известные нюансы
- `@keyframes shimmer` определён в `<style>` блоке в конце `<body>` (не в `<head>`)
- Инлайн-стиль `display:block;filter:...` на SVG-элементах — width контролируется через CSS-класс `.proj-svg`
- GitHub Pages может кешировать — для проверки изменений нужен жёсткий refresh
